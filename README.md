A terminal-based queue management engine designed to simulate real-world service flow. This project focuses on **sequential data processing** and **state management** within a CLI environment.

## 🛠️ Technical Stack
* **Language:** Python 3.x
* **Core Logic:** Lists, While-loops, and Conditional Branching
* **Interface:** Command Line Interface (CLI)

## 🚀 Key Features
* **Dynamic Token Generation:** Automatically initializes a daily sequence of service IDs.
* **Linear Queue Processing:** Implements a "Next-in-Line" logic triggered by user interaction.
* **Input Handling:** Sanitized terminal inputs to prevent execution breaks.

## 🧠 Learning Objectives
This repository serves as a practical implementation of **Computer Science fundamentals**, specifically:
1.  **Memory Management:** How Python handles list indices during iteration.
2.  **User Experience (UX) in CLI:** Creating an intuitive flow for the end-user without a GUI.
3.  **Clean Code:** Writing readable and maintainable Python scripts.
